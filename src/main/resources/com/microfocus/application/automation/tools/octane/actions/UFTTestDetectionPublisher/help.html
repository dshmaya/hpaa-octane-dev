This build step discovers UFT tests and external data table in the SCM repository and synchronizes corresponding entities to ALM Octane.
Discovery can done in two ways
  • Full sync - Cloned SCM repository is scanned and all found tests and data tables are synchronized to ALM Octane.
  • ChangeSet sync - Only commits that are related to the build are scanned and only affected entities are synchronized.
By default, full sync is done for the first run of the job, and after that only ChangeSet sync is done. If you need to force full sync - define in the job a boolean parameter "Full sync" and execute the job with this parameter is checked.
If you need to assign a discovered test in ALM Octane to a specific test runner, define a string parameter "Test Runner ID" and assign to it the ID of the relevant test runner.
